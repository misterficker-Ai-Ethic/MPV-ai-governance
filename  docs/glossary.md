# Glossary

This glossary defines key terms used in this AI governance project.

| Term | Definition |
|---|---|
| AI Governance Assessment | A structured record used to document and review an AI use case from a governance perspective. |
| Company | The organization that uses or evaluates the documented AI use case. |
| Use case | A specific situation in which an AI system is used to support a task, workflow, or decision. |
| Department | The business unit or team responsible for the documented use case. |
| Purpose | The intended goal or business reason for using the AI system in the documented use case. |
| AI system | The AI tool, model, or application used in the documented use case. |
| Data source | The origin of the data used by the AI system, such as internal documents, databases, or external datasets. |
| Personal data | Any information that relates to an identified or identifiable person. |
| Human review | A process in which a person checks, approves, or oversees AI-generated output. |
| Decision impact | The degree to which AI-supported output can influence a decision, action, or outcome. |
| Risk level | The overall assessment of how sensitive, critical, or high-risk a use case is from a governance perspective. |
| Recommended measures | Suggested actions to improve governance, oversight, transparency, or risk control for a use case. |
| Data dictionary | A document that describes the fields, meanings, and expected values used in the project data model. |
| JSON Schema | A machine-readable file that defines the allowed structure and validation rules for JSON records. |
| Example record | A sample JSON document that shows how a valid assessment entry should be structured. |
